## 关键代码事实（实现前必读）

- StepAudio delta 是**单字符粒度**，但 `_aggregate_stepaudio_deltas`（`transcript.py:816`）聚合成句级 chunk（≤40 字）后**逐字时间戳已丢失**。因此显示块重切只能在**句内按字符数比例**分配 `start/end`。
- `transcript.srt` 由 `cli.py:182` 对**全量 chunk** 调 `export_srt` 生成 → 过滤/重切**不得**进入 `export_srt` 本体或全量路径。
- 短视频字幕唯一生成点在 `export.py:118`：`export_srt(_slice_chunks_for_clip(chunks, clip_start, clip_end), subtitle_path)`。过滤与重切挂在这里。
- `_slice_chunks_for_clip`（`export.py:155`）已把 chunk 时间重基到 clip 的 0 起点，且 `clip_start = max(0, candidate.start - buffer_start)`，与 `clip_segment` 的裁剪起点一致 → 烧录用的 SRT 天然是 0 基准。
- `clip_segment`（`media.py:21`）当前用**输出侧** `-ss start -to end` 重编码（libx264）。烧录需要 0 基准时间轴，须改为**输入侧** seek（`-ss start -i input -t duration`），但**仅在传入字幕时**走该分支，无字幕时保持原命令不变（保护 `test_media_report_cli.py` 的精确断言）。
- live 短视频是**单段连续裁剪**（不走 `concat_videos` 去内部静音），时间轴无错位风险。

## 任务 2：语气词过滤纯函数

文件：新增 `video_auto_editor/subtitle.py`（字幕后处理纯逻辑，provider 无关）。

函数：`filter_filler_words(text: str, filler_words) -> str`
- 按标点与空白把 `text` 切成 token（保留分隔信息以便重组），分隔符集合至少含 `，。！？、；：…,!?;: ` 与空白。
- **整 token 命中规则**：仅当某 token 去除首尾空白后**完全等于**某个语气词（或完全由语气词字符构成的连续重复，如「嗯嗯」「啊啊啊」）时，丢弃该 token。
- **不**对多字符实词 token 做首尾语气词剥离——保证「好啊」「是啊」「这样吧」不被误伤（与设计共识 8 一致）。
- 重组后清理因删词产生的悬挂/重复标点与首尾标点（如开头残留的「，」）。
- 全部被删时返回空字符串（交给调用方丢弃该块）。

测试：`tests/test_subtitle.py`
- 独立成块：`"嗯。"` → `""`；`"嗯嗯"` → `""`。
- 句中夹带：`"嗯，今天我们讲愉悦技术"` → `"今天我们讲愉悦技术"`；`"那个，呃，重点是"` 中 `呃` 删除、`那个`（口头禅，不在词表）保留。
- 黏连不误伤：`"好啊"` → `"好啊"`；`"是啊，没错"` → `"是啊，没错"`。
- 多处与残留标点：`"啊，对，嗯，就这样"` → `"对，就这样"`（首尾及重复逗号清理）。
- 空词表：原样返回。

✅ git commit：`feat(subtitle): 实现按 token 边界的纯语气词过滤`

## 任务 3：字幕显示块重切纯函数

文件：`video_auto_editor/subtitle.py`（续）。

函数：`resegment_chunk(chunk, max_chars_per_line, max_lines) -> list[TranscriptChunk]`
- 块容量 `block_cap = max_chars_per_line * max_lines`（默认 30）。
- 文本 `≤ block_cap`：单块返回，但若 `> max_chars_per_line` 则在中点附近、优先靠近标点处插入换行（SRT 中以 `\n` 表示，烧录由 libass 渲染为多行）。
- 文本 `> block_cap`：优先按句末/逗号切分，凑成 `≤ block_cap` 的块；不得不切长 token 时按字符切。每块的 `start/end` 在原 chunk `[start,end]` 内**按字符数比例**分配，相邻块时间不重叠、不留缝（前块 end == 后块 start）。
- 配套 `resegment_chunks(chunks, ...)`：对列表逐块展开并保持时间有序。

测试：`tests/test_subtitle.py`（续）
- 短句不切：≤15 字单行、无换行。
- 中句换行：20 字 → 单块两行，行长均 ≤15。
- 长句多块：40 字 → 多块，每块 ≤30 字，时间按比例切分且首块 start==原 start、末块 end==原 end、块间无缝。
- 比例时间断言：等长两段切分后时间中点落在块边界附近（容差断言）。

✅ git commit：`feat(subtitle): 实现过长字幕按字符比例重切为显示块`

## 任务 5：`clip_segment` 支持烧录字幕

改动：给 `clip_segment` 增加可选关键字参数 `subtitle_path=None`：

- `subtitle_path` 非空 → 走烧录分支：
  - 改用**输入侧 seek**：`ffmpeg -y -ss <start> -i <input> -t <duration> -vf "subtitles=<escaped>:force_style='<style>'" -c:v libx264 -crf .. -preset .. -c:a aac -b:a .. <out>`，保证输出 PTS 0 基准、与 0 基准 SRT 对齐。

  - 路径转义：新增 helper 对 `subtitles=` 的文件名转义 `\`、`:`、`'`（libass filter 语法）。
- 从 `config` 读取 `subtitle_font/_size/_outline/_margin_v`。

测试：`tests/test_media_report_cli.py`
- 现有 `test_clip_segment_builds_existing_ffmpeg_command`（无字幕）保持通过，**不改**。
