# 短视频字幕大模型优化 — 实现计划

> 关联设计：[ADR 0011](../adr/0011-optimize-clip-subtitles-with-llm-under-subsequence-constraint.md)、`CONTEXT.md`（字幕优化 / 字幕优化模型 / 字幕显示块 / 语气词过滤 / 烧录字幕 / 处理缓存 / 标准交付物）。

## 目标

解决短视频字幕「一句话被切断」「语气词单独成行」：在导出阶段，对每条 clip 窗口的连续文本调用**字幕优化模型**，在**子序列约束**（只删字、只断句、不增改字）下做语义级删词与切分，生成字幕显示块；显示块按 ASR 逐字时间精确对齐。优化失败/不可用则回退规则路径、不烧录、标人工复核。

## 接入形态

```
ASR deltas(逐字) ──聚合──> TranscriptChunk[+char_spans] ──缓存──┐
导出时(每条 clip):  clip 窗口 chunks                            │
   ├─ 拼接窗口文本 + 逐字时间映射(char_spans, 缺失则按 chunk 比例兜底)
   ├─ 字幕优化模型(纯文本分块) ──校验子序列──> 块边界决策
   │      ├─ 成功: 两指针对齐回逐字时间 → 显示块 → 烧录 + 旁挂优化 SRT
   │      └─ 失败/不可用: filter_filler_words + resegment_chunks → 不烧录 + 旁挂规则 SRT + 标人工复核
   └─ 独立缓存: 签名=窗口文本+模型+提示+删词尺度+切分参数; 只存块字符串, 时间用时重算
```

## 设计决策摘要

| # | 决策 | 结论 |
|---|------|------|
| Q1 | LLM 改写权限 | 子序列：可删可断、不可增改字 |
| Q2 | 作用范围 | 每条 clip 窗口跨 chunk 一次调用 |
| Q3 | 时间锚点 | 逐字精确对齐（保留 ASR 逐字时间） |
| Q4/4b | 失败回退 | 强依赖；失败则不烧录、仍导出视频+旁挂规则 SRT、标人工复核 |
| Q5 | 删词尺度 | 语气词 + 语境口头禅/口吃重复/无意填充 |
| Q6 | 输出契约 | 纯文本分块（每块一行），不走 JSON |
| Q7 | 缓存 | 独立缓存、按字幕输入签名、不污染 ASR/评审缓存 |
| Q8 | 模型 | 复用评审模型接口，可独立指定模型 id |
| 默认值 | `subtitle_optimization_enabled` | `True`（export 已被同一 key 卡住，主流路径零回归；`False` 为逃生舱） |

## 风险与守门

1. **T1.3 动缓存格式**是唯一可能踩既有测试的点：先全量回归，新增键须向后兼容；若某用例 pin 精确 JSON，调整**生产侧**表示而非测试。
2. **子序列禁补标点**可能抬高失败率→失败软降级（旁挂规则 SRT，文本质量同今天）+ 报告可观测；上线后看「待人工复核」比例，必要时回 ADR 0011 微调（如标点白名单）。
3. **非 StepAudio 识别路径**无逐字时间→`char_spans=None`→比例兜底，时间精度退化但不阻塞。

## 验收标准

- 有 key 主流路径：导出 clip 默认带经优化的烧录字幕，无「半句被切断」「语气词单独成行」；旁挂 SRT 为同款显示块。
- 优化失败的 clip：视频照常导出、不烧录、旁挂规则 SRT、报告标「待人工复核」。
- `transcript.srt` 不变，仍忠实完整、不删语气词。
- ASR/评审缓存签名不受字幕优化配置影响；字幕优化缓存命中可免重复调用。
- 全量既有测试保持绿，新增测试覆盖上述路径。
