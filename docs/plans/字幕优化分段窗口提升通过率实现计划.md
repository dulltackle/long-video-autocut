# 字幕优化分段窗口提升通过率 — 实现计划

> 关联设计：[ADR 0011](../adr/0011-optimize-clip-subtitles-with-llm-under-subsequence-constraint.md)（本计划修订其 Q2「作用范围」与 Q6「提示词形态」）、`docs/plans/字幕优化实现计划.md`、`CONTEXT.md`。

## 背景与根因

`video-auto-editor live` 真实端到端跑通后发现：5 条短视频里 **4 条画面无烧录字幕**。诊断链：

1. 烧录已开启（`burn_subtitles` 默认 True），但每条 clip 烧录前先跑 LLM 字幕优化（`step-3.7-flash`）。
2. 优化器要求模型输出是原文的**子序列**（只删字断句、不增改字），由 `subtitle_align.validate_and_align` 两指针校验。
3. `step-3.7-flash` 违反约束：① 用空格当显示块分隔符（增字）；② 改写/造词/语序重排（如「千万不要」→「千万要注意避孕」、造词「法拉利核武器」）。
4. 校验失败 → 状态 `OPTIMIZE_FAILED` → `export.py` 刻意**抑制烧录**、回退旁挂规则 SRT、标人工复核。5 条里仅 clip 2 凑巧产出干净子序列被烧录。

## 实验结论（已用真实素材验证）

三个杠杆叠加，子序列通过率 **1/5 → 5/5**：

| 阶段 | 通过率 |
|---|---|
| 原始（旧提示·high·temp0.2·整窗口） | 1/5 |
| 新提示（抽取式 + 禁空格 + few-shot） | 2/5 |
| + `reasoning_effort=low` + `temperature=0` | 3/5 |
| + **窗口字符预算 ≈ 100 分段逐组请求** | **5/5** |

- 提示词改写消灭「增字/空格」类违规；低推理 + 零温度抑制改写冲动；短窗口把「语序漂移」概率压到极低（文本越短模型越没空间重组）。
- 代价：单条 clip 请求数 1 → 4~6 次；但优化器按**子窗口文本签名缓存**，同一窗口只真实请求一次、跨运行复用，属一次性成本。

## 目标

把「分段窗口」从实验脚本落进生产：超过字符预算的 clip 窗口按 chunk 贪心分组，逐组请求 + 校验对齐 + 拼接显示块；任一组失败仍整体判 `FAILED`（保持现有失败抑制烧录策略）。配合已改进的抽取式提示词与本地调优参数，使该素材 5 条全部成功优化并烧录。

## 设计要点

```
optimize_window(window_chunks):
   groups = _split_window(window_chunks, window_max_chars)   # 贪心按字符预算分组，单 chunk 超额则独占一组
   all_blocks = []
   for group in groups:
       text, char_times = build_window(group)                # group 内 chunk 已是 clip 相对时间，跨组时间天然连续
       block_lines = _block_lines_cached(text)               # 复用 _read_cache → _resolve_block_lines → _write_cache
       if block_lines is None: return None                   # HTTP/解析失败 → 整体 FAILED
       aligned = validate_and_align(text, char_times, block_lines, max_chars_per_line, max_lines)
       if aligned is None: return None                       # 非子序列 → 整体 FAILED
       all_blocks.extend(aligned)
   return all_blocks
```

- **缓存粒度**：按子窗口文本签名（`_cache_signature` 已含 text + model + base_url + 切分参数 + `PROMPT_VERSION`）。`window_max_chars` **不进签名**（它只决定如何切，切出的 text 已在签名里）。
- **默认值**：`subtitle_optimization_window_max_chars = 100`。该默认下，既有短窗口单测（窗口文本 < 100 字）仍是单组、单次请求，行为不变、零回归。
- **修订 ADR 0011**：Q2「每条 clip 窗口跨 chunk 一次调用」改为「按字符预算分段、逐组调用」；Q6 提示词由「优化/整理」改为「抽取式删减（禁空格 + few-shot）」。
- **本地调优参数**：`reasoning_effort=low` + `temperature=0` 已写入 `config.local.json`（被 `.gitignore` 忽略，不进提交）；不改 `config.py` 全局默认，避免影响其它模型/场景。
