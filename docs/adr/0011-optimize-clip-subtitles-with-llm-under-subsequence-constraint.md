# 用大模型在子序列约束下优化短视频字幕

> 修订：[ADR 0012](0012-segment-subtitle-optimization-window-to-improve-subsequence-pass-rate.md) 修订本文「作用范围」与「输出契约/删词尺度」对应的提示词形态——把「每条 clip 窗口跨 chunk 一次调用」改为「按字符预算分段、逐组调用」，提示词改为抽取式删减（橡皮擦隐喻、禁空格、few-shot）。

规则管线（句级切分 + 纯语气词词表 + 按字符比例分配时间）会把一句话切断、让落单的口头禅单独成块，可读性差。我们决定在转写之后、导出阶段，对每条短视频窗口的连续文本调用字幕优化模型，做语义级断句与删词，生成字幕显示块。为既提升质量又能精确对齐，约束 LLM 的输出必须是原文的**子序列**（可删字、可断句，不可增改字），从而每个显示块都能按字符两指针对齐回 ASR 逐字时间。

## Considered Options

- **改写权限**：选「子序列」而非自由改写——自由改写质量上限更高，但时间戳会失去字符锚点，且引入 ASR 之外的事实风险。子序列让逐字精确对齐成为确定、无歧义的两指针计算。
- **作用范围**：选「每条 clip 窗口跨 chunk 一次调用」而非单 chunk 内或整场一次性——「被切断」的根因之一是 ASR 在 `_STEPAUDIO_MAX_CHUNK_CHARS` 处没等到标点就硬断句，只有跨 chunk 才能把半句合回完整句再按语义重切。
- **时间锚点**：选「逐字精确对齐」而非沿用 chunk 级比例分配——比例分配在 LLM 合并长句后会漂移一两秒，烧录字幕与语音肉眼可见错位。代价是把 ASR 逐字时间贯穿到聚合之后。
- **删词尺度**：删语气词，并借语境删口头禅、口吃重复、无意义填充（规则词表不敢碰「那个/就是」怕误删，LLM 有语境可精准定位），但保留所有承载语义的词。
- **输出契约**：选「纯文本分块（每块一行）」而非 JSON——flash 类模型返回不合规 JSON 是已知痛点（见处理缓存的字幕优化签名设计动机）；纯文本去换行拼接后，子序列校验与逐字对齐都是 O(n) 两指针，一步到位。
- **失败回退**：把字幕优化作为强依赖。调用失败、超时、无 API Key、或输出违反子序列/无法对齐时，该条短视频**不烧录字幕**，仍导出视频并旁挂规则生成的 SRT（语气词过滤 + 规则重切），在拆条报告标注待人工复核。即「画面字幕」这一层降级，但交付不丢。
- **模型选型**：复用评审模型的 provider/接口/凭据配置，允许单独指定字幕优化的模型 id 与提示，新增「字幕优化模型」概念。

## Consequences

- 字幕优化位于转写之后、缓存为独立命名空间（按 clip 文本窗口、模型、提示、删词尺度、切分参数签名），不污染语音识别与主题评审缓存（见 ADR 0009、0010）。
- 逐字精确对齐要求把 ASR 逐字时间贯穿到句级聚合之后，涉及 chunk 模型、分片偏移、重叠合并与缓存序列化，**处理缓存格式因此变更、签名需刷新**。
- 子序列禁止增字，意味着 LLM 连补标点都不允许；若模型习惯性顺滑加标点会触发校验失败、走人工复核回退，需在提示中明确禁止增改字、并接受一定回退率。
- 语气词过滤与规则重切从主路径转为字幕优化的确定性兜底，继续保障字幕优化不可用时仍有可读旁挂字幕。
- `transcript.srt` 不经字幕优化，保持忠实完整、不删语气词。
- 字幕优化默认开启（`subtitle_optimization_enabled=True`）。之所以敢把一个 LLM 硬依赖默认开启：短视频导出本就被同一个 `STEPFUN_API_KEY` 通过主题评审卡住——无 key 时默认 `review_status=unreviewed` 且不导出任何 clip，故主流发布就绪路径要么有 key（优化可用）、要么没有 clip 可优化，零回归。唯一回归点是「无 key + 显式 `allow_unreviewed_export`」这一用户主动接受未评审质量的兼容路径，可用 `subtitle_optimization_enabled=False` 逃生。默认值本身一行可翻，不属难回退决策。
