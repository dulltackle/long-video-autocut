# Long Video Autocut

这个上下文描述长视频自动粗剪与直播拆条中的核心业务语言，帮助后续设计围绕同一组概念展开。

## Language

**CLI 底座**:
负责直播拆条核心处理流程的可安装命令行能力，包括媒体处理、语音识别、候选生成、主题评审调用、边界补救、导出选择和标准交付物生成。
_Avoid_: 命令行工具、底层工具、video_auto_editor

**skill 调度器**:
围绕 CLI 底座提供上层协作流程的调度能力，负责环境预检、收集课程上下文、组装参数、调用 CLI、解释结果和提示修复。
_Avoid_: skill、上层脚本、自动化助手

**直播拆条**:
把一场较长的直播视频拆成多条可独立观看和传播的短视频。
_Avoid_: 直播切片、直播剪辑、长视频拆分

**候选片段**:
从原始视频中识别出的、可能适合导出为短视频的一段内容。
_Avoid_: 窗口、片段窗口、素材段

**相邻候选批次**:
按原视频时间顺序组织的一组候选片段，用于让主题评审理解前后文和边界关系。
_Avoid_: batch、评审批次、候选集合

**主题片段**:
围绕一个相对完整教学点或营销表达点的连续内容单元。主题变化是划分边界的首要依据，明显停顿或转场用于辅助校准，目标时长只作为约束。
_Avoid_: 主题窗口、知识点片段、语义片段

**同主题系列**:
同一个较长主题被拆成多条发布就绪短视频后形成的连续短视频集合。系列中的每条短视频都应有相对独立的小结论或小步骤。
_Avoid_: 分集、连续切片、上下集

**主题判定**:
判断一段内容是否围绕同一主题、是否适合作为主题片段的业务判断。
_Avoid_: 语义分类、LLM 分析、智能识别

**主题评审**:
对候选片段进行主题完整度、学习价值、传播价值和导出建议的结构化判断。
_Avoid_: AI 选择、智能筛选、模型打分

**评审模型**:
用于执行主题评审的文本推理模型。默认评审模型使用 StepFun Chat，底层配置应允许替换为兼容接口。
_Avoid_: LLM、Chat 模型、大模型

**导出选择**:
根据主题评审结果和业务约束确定最终导出的短视频集合。
_Avoid_: LLM 决策、自动挑选、最终打分

**人工复核**:
系统认为内容有价值但无法完全满足发布就绪标准时，对该短视频或候选片段提出的人工确认要求。
_Avoid_: 人工审核、待确认、人工处理

**淘汰原因**:
候选片段没有成为发布就绪短视频的业务原因。淘汰原因用于报告解释和人工二次处理。
_Avoid_: 过滤原因、失败原因、拒绝理由

**短视频**:
从长视频中导出的单条成品视频，时长较短，适合学习、复盘或传播。
_Avoid_: clip、切片、视频段

**发布就绪短视频**:
几乎不需要人工返工即可交付或发布的短视频。发布就绪要求主题完整、开头不突兀、结尾不截断、表达流畅、音画边界自然，并带有可直接使用的标题、摘要、字幕和元数据；默认时长为 1 到 5 分钟，优先接近 3 分钟。
_Avoid_: 可用片段、优质切片、成品片段

**边界补救**:
当候选片段主题有价值但开头或结尾不自然时，向相邻内容或静音边界扩展以尝试恢复上下文完整性的处理。
_Avoid_: 边界修正、前后扩展、补上下文

**主题覆盖**:
最终短视频集合对原直播中有效主题的覆盖程度。主题覆盖服从发布就绪质量和主题完整性，不为了提高覆盖而导出低质量短视频。
_Avoid_: 素材利用率、覆盖率、尽量多切

**转写文本**:
原始视频语音内容转换成的带时间信息文本。
_Avoid_: 字幕文本、识别文本、稿件

**语音识别**:
把原始视频中的语音内容转换为转写文本的能力。默认语音识别服务使用 stepaudio-2.5-asr，其他识别方式只作为可选替代。
_Avoid_: ASR、Whisper、音频识别

**识别分片**:
为了进行稳定语音识别而从原始视频音频中切出的连续音频片段。识别分片用于降低长音频请求和重试成本，合并后形成整场直播课的转写文本。
_Avoid_: 音频切片、转写分片、ASR 分片

**课程上下文**:
用户可选提供的原视频以外信息，用于帮助理解直播课内容和交付偏好。课程上下文可以包括课程主题、目标客户画像、讲师或品牌名称、禁用词、希望重点保留的主题和不希望导出的内容类型；传给 CLI 底座时使用 JSON 文件。
_Avoid_: 提示词、补充信息、配置说明

**标准交付物**:
一次直播拆条完成后交付给用户的文件集合，包括短视频、字幕、转写文本、元数据和拆条报告。
_Avoid_: 输出结果、产物、导出文件

**拆条方案**:
在真正导出视频前形成的预期短视频清单和相关判断，包括候选片段、主题评审结果、预计导出项和人工复核提示。
_Avoid_: dry-run、预览、分析结果

**未评审方案**:
缺少主题评审结果的拆条方案。未评审方案只能用于查看基础候选和诊断流程，不代表短视频已达到发布就绪标准。
_Avoid_: 基础方案、规则方案、无模型结果

**处理缓存**:
直播拆条过程中复用的中间结果，包括识别分片转写、整场转写文本和主题评审结果。处理缓存必须能区分会影响结果的视频、配置和课程上下文变化。
_Avoid_: cache、中间文件、临时结果
